# Organizations Feature Module

This module manages the lifecycle, status, and configuration of organizations within the Nexo Admin platform.

## 🏛️ Architecture Overview

The feature follows a decoupled **Service-Hook-Component** architecture:

```mermaid
graph TD
    Page[OrganizationsPage.jsx] --> Hook[useOrganizationsPage.js]
    Hook --> Service[orgService.js]
    Service --> CoreAPI[@api/client.js]
    Hook --> CommonUI[OrganizationsTable.jsx]
    CommonUI --> AtomicUI[Card, Button, DataTable]
```

## 📂 Directory Structure

- **`services/`**: Isolated API calls for organizations. Uses `@api` client.
- **`hooks/`**: Headless logic for pages and complex views.
  - `useOrganizationsPage.js`: Main list orchestration.
  - `useOrganizationDetail.js`: Detail view orchestration.
  - `useCreateOrganizationPage.js`: Form orchestration.
- **`components/`**: Domain-specific UI fragments.
  - `OrgHero.jsx`: High-impact summary header.
  - `OrgStats.jsx`: Resource usage metrics (using `@components/ui/Card`).
- **`constants/`**: Mock data, table columns, and form options.
- **`styles/`**: Exclusive CSS for organizational layouts.

## 🚀 Standards Enforced

1.  **Headless Hooks**: All complex logic is extracted from JSX files.
2.  **Modular Cards**: Uses the `nx` variant for premium, interactive stat cards.
3.  **Clean Imports**: Utilizes project-wide aliases (`@core`, `@context`, `@api`).
